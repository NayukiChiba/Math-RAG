# experimentWebUI.py

## 概述
`scripts/experimentWebUI.py` 或 `scripts/experiments/experimentWebUI.py` 补充提供了一套可视化的界面使得开发者可以直接在这个网页系统操作和调整他们想要的参数实验配置而无需进文件改代码。并在执行完如 `runExperiments.py` 的一系列后得到包含不同搜索参数的回答效果双列展示或者是多种模型生成的准确性分数的统计比较图等提供图形前端和调度展示界面的组件工具。 

## 函数与接口机制介绍

### `_build_comparison_layout()`
利用 `Gradio` 的类或者是行内表格等特性声明：一侧放置原始系统的打分记录或问答内容；另一侧或下侧罗列当调校了新的搜索权重或者切换新基座后生成的应答结果组件界面的结构排版构造设计部分抽象逻辑方法机制。 

### `_start_experiment_from_ui(config_params)`
界面核心执行桥接响应组件函数点。捕捉浏览器界面传来的勾选项，滑块配置生成组装并作为执行请求推送到引擎后端执行 `runExperiments` 所相关跑批进程或启动并接回最终输出状态以回写在界面状态层使得人类看到过程结果反馈层回调接口响应机制逻辑封装对象工具设计等操作端点实现区。 

### `launch_experimental_dashboard(port: int = 7861)`
启动能够进行多系统参数对比评估操作大工作版大盘面板程序入口点启动监听并在给定网络监听位提供服务拉取控制。 

### `main()`
作为终端独立唤起和测试这个复杂分析页面的快速开关测试执行操作方法主层调用命令实现门面逻辑和入口实现封装点功能设计包等操作命令配置起始功能接入口调用操作机制。
