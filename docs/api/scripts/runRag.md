# runRag.py

## 概述
`scripts/runRag.py` 或 `scripts/pipelines/runRag.py` 作为一个综合性流水线调度脚本，负责把整个 Math-RAG 项目里的各类业务组件（如数据读取、预处理、建库检索到最后大模型产生答案）按业务逻辑串联成一系列阶段性启动项或者可以端到端（End-to-End）直接贯穿运行的应用流程执行器。

## 主要工作阶段

###  Phase 1: 数据入库
它可能涵盖检查配置的原始 `PDFs` 是否都被完整通过基于 `pix2text` 或者关联调用的脚本走过了一遍 OCR 化成为了对应的预处理结果与建立倒排和 Dense 索引存储介质中。如果发现落出空洞等缺失或者配置项为强制重新构建，则激活执行诸如 `buildCorpus.py` 或相应的依赖方法等。 

###  Phase 2: 加载向量引擎并启动查询
接收或轮询传入的命令测试输入数组，加载起组装含有混合查找引擎 `HybridRetriever` 之上的包装控制组件并将参数里的 Top-K 文档寻找并缓存截取回来待答复拼接等部分调度流实现处理中枢控制器逻辑位置设计执行。

###  Phase 3: 合并提示词呼叫远端
这部分的串线把以上两个主要大结构组交联给基于 `qwenInference.py` 及大模型对话生成接口并下发进行对话补全响应获取等业务衔接驱动流程链。

## 函数与接口机制概要
（取决于版本和更新情况：内部通常具有以下类似接口实现）：
- `_prepare_working_directory()`
- `_init_components(config)`
- `execute_qa_flow(query_string)`
- `main(args)`
允许借助形如 `--pipeline-mode all` `--input-query "高斯分布"` 等一系列高阶复合参数操控整一条数据业务跑批的控制。它是最贴近终端操作者用以观察和验收核心工作效果的一类包装演示脚手架层程序。
