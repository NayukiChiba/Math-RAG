# Math-RAG 文档目录说明

本文档用于说明 docs 目录内每个文件的职责、使用场景与维护方式。

## 文档文件作用

- plan.md
  - 项目层面的长期规划文档。
  - 关注研究目标、范围边界、里程碑、风险与后续路线。
  - 适合用于答辩汇报、论文方法章节的总体叙述。

- task.md
  - 工程层面的执行清单与进度文档。
  - 关注任务拆分、状态、验收标准、输入输出路径。
  - 适合用于周报、迭代管理、复现实验检查。

- project.md
  - 项目说明主文档（对外/对内都可用）。
  - 覆盖架构、目录职责、运行流程、常用命令、输出产物。
  - 适合作为新成员入门文档和论文实验部分的操作依据。

## 阅读顺序

1. 先读 project.md，快速理解系统怎么跑。
2. 再读 plan.md，理解项目为什么这样设计。
3. 最后读 task.md，查看当前做到哪里、下一步做什么。

## 维护约定

- 修改代码结构后，优先同步 project.md 的目录说明和命令示例。
- 任务状态变化时，只更新 task.md，不在其他文档重复维护进度。
- 里程碑或研究边界调整时，更新 plan.md。
- 文档中的路径一律使用项目根目录相对路径。
